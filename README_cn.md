# MYEDGENEWTAB

一个可自定义的 Microsoft Edge 新标签页，具有背景轮播和必应搜索功能（优化了图片切换时的白屏问题）。

**切换到英文版本：** [README.md](README.md)

## 版本核心区别

- **v1.2.4（当前版本）**：设置面板深海蓝暗色主题全面改版——蕾姆女仆配色、天蓝光晕交互、渐变标题与顶部彩条、重绘滑块/按钮/滚动条、SF Pro Display 字体引入、更深邃的毛玻璃抽屉。

- **v1.2.3（旧版）**：设置面板视觉与交互体验全面优化——更清晰的设计层级、更舒适的呼吸感、按钮与进度条加入弹性动画；修复折叠图标偶发不旋转、"恢复默认配置"后进度条未同步两个 bug。

- **v1.2.2（旧版）**：在搜索框外观设置中新增搜索框文字颜色选择器。

- **v1.2.1（旧版）**：在 v1.2.0 基础上进行了小幅 bug 修复和稳定性优化。

- **v1.2.0（旧版）**：新增完整设置面板——可直接在扩展内导入本地图片（通过 IndexedDB 存储）、外观与轮播配置持久化、设置分区可折叠，并优化了轮播的资源占用。

- **v1.0.0（旧版）**：需要手动在核心文件中编辑图片文件名。

## 功能特性

- **本地图片导入：** 通过设置面板直接从设备选择并导入图片，图片以 IndexedDB 存储在扩展内部，无需手动管理文件。

- **双图片来源：** 可随时在设置面板中切换"我的导入图片"和"内置图片库"（打包的 `images/` 文件夹）。

- **设置面板：** 右侧滑出抽屉，配置跨会话持久保存，包含搜索框外观、搜索按钮外观、轮播等可折叠分区。

- **搜索框自定义：** 通过滑块和颜色选择器调整位置（距顶部距离）、宽度、高度、圆角、背景透明度及**文字颜色**。

- **搜索按钮自定义：** 通过滑块和颜色选择器配置按钮颜色、字号、圆角及内边距（上下/左右）。

- **轮播自定义：** 设置切换间隔、过渡动画时长、随机顺序、暂停轮播、页面隐藏时暂停。

- **恢复默认配置：** 一键将所有设置恢复为默认值（v1.2.3 修复了进度条未同步的小问题）。

- **设置面板体验优化（v1.2.3）：** 统一的设计系统 token、柔和的色板、按钮与进度条加入弹性动画（按下回弹），分区折叠图标始终精准同步。

- **自动图片列表生成：** 批处理文件扫描 `images/` 文件夹并生成 `image-list.json`（支持中文/带空格的文件名），供内置图片库使用。

- **增强的文件名兼容性：** 支持带空格和编码 `%20` 字符的文件名。

- **平滑背景轮播：** 预加载图片以消除切换时的白屏（过渡时长可配置）。

- **必应搜索集成：** 功能完整的搜索框（支持回车键/按钮搜索）。

- **错误处理：** 图片/JSON 文件缺失时显示清晰警告，图片加载失败时有备用样式。

## 支持环境

- Microsoft Edge（基于 Chromium） - [下载链接](https://www.microsoft.com/edge)

- Windows 操作系统（批处理文件仅适用于 Windows）

- Node.js（批处理文件运行 `generate-image-list.js` 所需） - [下载链接](https://nodejs.org/)

- Git（用于克隆/更新本地仓库） - [下载链接](https://git-scm.com/downloads)

> **注意：** Node.js 和 Git 仅在使用自动生成图片列表功能或通过 Git 克隆仓库时才需要。对于大多数用户，设置面板的导入功能已完全足够。

## 安装方法

### 选项 1：从发布版下载（推荐普通用户）

1. 从 [发布页面](https://github.com/YourUsername/MyEdgeNewTab/releases) 下载最新版本文件夹

2. **加载到 Edge：** 打开 `edge://extensions/` → 启用开发者模式 → 点击"加载已解压的扩展程序" → 选择下载的文件夹

### 选项 2：克隆仓库到本地（适用于开发者/高级用户）

```bash
git clone https://github.com/YourUsername/MyEdgeNewTab.git
cd MyEdgeNewTab
```

然后通过 `edge://extensions/` 加载文件夹（需启用开发者模式）。

## 使用方法

### 导入自己的图片（推荐）

1. 打开新标签页 → 点击右上角 **⚙** 按钮打开设置面板

2. 在**背景图片**分区中，确认选中"导入我的图片"

3. 点击 **+ 选择本地图片**，从设备中选择一张或多张图片

4. 图片将保存在扩展内部（IndexedDB），跨会话持久保留

5. 如需删除单张图片，将鼠标悬停在预览缩略图上并点击 **×**；如需删除全部，点击**清空**

### 使用内置图片库

1. 将图片（仅限 PNG/JPG/JPEG/WEBP 格式）放入 `images/` 子文件夹

2. 确认已安装 Node.js（运行 `node --version` 检查）

3. 双击 `generate-image-list.bat`（仅限 Windows）生成 `image-list.json`

4. 打开设置面板 → 在背景图片分区中选择"内置图片库"

### 使用自定义新标签页

- 新的 Edge 标签页将自动轮播图片

- 使用搜索框：输入查询 → 按回车键或点击搜索按钮运行必应搜索

- 点击 **⚙** 随时打开设置面板，调整外观、轮播行为和图片来源

## 自定义配置

所有视觉和行为设置均可在设置面板（⚙ 按钮）中完成，更改自动保存并跨会话持久保留。也可通过**🔄 恢复默认配置**按钮一键重置所有设置。

对于偏好直接编辑代码的高级用户：

### 调整轮播间隔

编辑 `script.js`，修改 `DEFAULT_SETTINGS` 对象：

```javascript
carouselIntervalSec: 4, // 修改为所需的秒数
```

### 自定义搜索框

编辑 `styles.css`，覆盖 CSS 变量：

```css
:root {
  --search-width: 600px;
  --search-radius: 30px;
  --search-text-color: #000000;
}
```

## 重要注意事项

- **导入的图片**存储在 IndexedDB 中，在浏览器配置文件内持久保留，重新加载扩展后无需再次导入；但如果卸载扩展或清除浏览器数据，图片将丢失。

- **内置图片库：** 在 `images/` 文件夹中添加/删除/替换图片后，必须重新运行 `generate-image-list.bat` 以更新 `image-list.json`。

- **图片格式：** 仅支持 PNG/JPG/JPEG/WEBP。

- **首次加载：** 图片在首次加载时会被缓存，后续切换无延迟。

- **Git 同步注意事项：** 如果修改了本地文件，运行 `git pull` 前请先备份。

## 故障排除

| 问题                   | 解决方案                                                               |
| -------------------- | ------------------------------------------------------------------ |
| "扫描图片失败"警告           | 重新运行 `generate-image-list.bat` 以生成 `image-list.json`               |
| "images 文件夹内无图片"警告   | 向 `images/` 文件夹添加有效图片 → 重新运行批处理文件                                  |
| 双击批处理文件无反应           | 检查是否安装了 Node.js，在命令提示符中运行 `node --version` 确认                      |
| 轮播时出现白屏              | 检查图片路径/文件名是否有误；确保图片格式有效（默认已启用预加载）                                  |
| 搜索框无法使用              | 验证 `script.js` 是否完整（不要修改 search-input/search-button 的 ID）→ 刷新标签页   |
| 重装扩展后导入的图片丢失         | 属于预期行为——IndexedDB 与扩展安装绑定，重新导入图片即可                                 |
| 设置面板中内置图片库为空         | 先运行 `generate-image-list.bat`，然后重新加载扩展                             |
| 搜索框文字颜色设置不生效         | 请确认版本为 v1.2.2 或更高；可尝试点击"恢复默认配置"后重新设置                              |
| 恢复默认配置后进度条卡在旧位置       | 请升级到 v1.2.3 或更高版本                                                       |
