## 折叠 ▶ 隔次旋转 bug（已修复）
- 文件：`script.js` L352-364（`section-toggle` 的 click 处理）
- 根因：`button.classList.toggle('expanded', !isHidden)` 写反了。`!isHidden` 是"展开后取 false"，所以展开时图标**不旋转**，折叠时才旋转——视觉上就是"展开时不动、折叠时才动"，循环起来像"隔两次才正常"
- 修复：把变量名改成 `willExpand = contentEl.hidden`，用 `willExpand` 直接当 `expanded` 类的开关
- 经验：布尔状态切换前，先想清楚"toggle(class, condition) 的 condition 是要表达目标态还是当前态"，目标态更安全

## 恢复默认配置"不管用" bug（2026-06-14）
- 文件：`script.js` L552-575 `syncUi` 函数
- 根因：之前给 range 进度条加 `--range-progress` CSS 变量做"已滑过部分渐变染色"，只在 `input` 事件 + 初次注册时同步。但 `reset → syncUi` 用 `rangeEl.value = ...` **JS 改 input.value 不触发 input/change 事件** → `--range-progress` 仍是旧值 → 进度条视觉上停在旧位置，看起来像 reset 失败
- 修复：在 `syncUi` 遍历 rangeIds 时，每个 range 都重新算一次 `--range-progress` 并写入行内样式
- 经验：所有"JS 改 input.value → 期望 UI 同步视觉状态"的场景，都要**手动同步**由 input 事件维护的派生状态（CSS 变量、ARIA 属性、文本标签等），不能依赖事件链
